# 渲染管线

本文档描述当前代码里真实存在的渲染路径，而不是理想化设计。

## 当前渲染模型

当前渲染器是单线程、Deferred 默认路径的简化实现，核心阶段如下：
- 游戏侧 `Primitive` 通过 `CreateSceneProxy()` 创建仅包含实例描述的 `SceneProxy`
- 游戏侧 `LightComponent` 通过 `CreateLightSceneProxy()` 创建方向光或点光的渲染镜像
- `FScene::AddPrimitive` 在注册阶段触发资源准备，并把可用 Proxy 注册到场景
- `FScene::AddLight` 把 `FLightSceneProxy` 注册到场景光源列表
- `FRenderResourceManager` 维护静态网格资产级 `RenderData` 缓存、材质贴图缓存与共享 Pipeline
- `FSceneRenderer` 调度当前 `IRenderPath`
- 当前默认 `FDeferredRenderPath` 通过 `FMeshPassProcessor(BasePass)` 从各个 Proxy 收集 `FMeshDrawCommand`
- `FDeferredRenderPath` 先执行 GBuffer Pass，把 Albedo、WorldNormal 与 Depth 写入离屏 `RHIRenderTarget`
- `FDeferredRenderPath` 再执行 Lighting Pass，通过全屏三角形采样 GBuffer，并从 `FScene::GetLights()` 读取方向光与点光累加光照
- 对 DrawCommand 做排序，减少状态切换
- 通过 `RHICommandBuffer` 提交命令
- 最终由 `RHIOpenGL` 执行实际图形 API 调用

## 关键类型

### `FScene`
职责：
- 作为 `IRenderScene` 的默认实现，提供 `Add/Update/RemovePrimitive`
- 提供 `Add/Update/RemoveLight`，维护当前活跃的 `FLightSceneProxy`
- 保存当前活跃的 `FPrimitiveSceneInfo` 与 `FViewInfo`
- 在 Primitive 注册阶段触发 Proxy 资源准备

### `FRenderResourceManager`
职责：
- 维护 `TStaticMesh -> FStaticMeshRenderData` 的资产级缓存
- 维护 `StaticMesh + MaterialIndex -> BaseColor Texture` 缓存
- 提供默认白纹理与默认采样器
- 维护静态网格共享 Shader/Pipeline
- 将 `FStaticMeshSceneProxy` 的资产描述解析为可绘制资源引用

### `FPrimitiveSceneInfo`
职责：
- 作为 Primitive 注册到场景后的运行时节点
- 绑定 `FPrimitiveComponentId`（稳定身份）与 `FPrimitiveSceneProxy`
- 作为 `FScene` 的基础存储单元，避免场景直接持有裸 Proxy 映射

### `FStaticMeshRenderData`
职责：
- 持有资产级 GPU 数据（统一 VertexBuffer + 统一 IndexBuffer）
- 记录每个 Section 的索引范围（`FirstIndex` + `IndexCount`）
- 被多个 `FStaticMeshSceneProxy` 共享引用，避免重复上传网格数据

### `FPrimitiveSceneProxy`
职责：
- 作为游戏侧 `PrimitiveComponent` 的渲染镜像基类
- 持有实例级渲染状态（例如世界变换）
- 对外提供绘制命令生成入口

### `FStaticMeshSceneProxy`
职责：
- 持有 `TStaticMesh` 资产引用（实例描述）
- 在渲染场景注册阶段由 `FRenderResourceManager` 注入共享 `RenderData`
- 为各 Section 生成绘制命令
- 在命令中输出 `PipelineKey`（而不是 RHI Pipeline 指针）
- 不直接创建或销毁 GPU 资源，也不负责资源缓存查询

### `FLightSceneProxy`
职责：
- 作为游戏侧 `LightComponent` 的渲染镜像
- 保存 Forward/Deferred 光照需要的类型、颜色、强度、方向、位置与衰减半径
- 不持有 GPU 资源

### `FMeshDrawCommand`
职责：
- 打包一次绘制所需描述（`FPipelineKey` / VB / IB / StaticMeshAsset / MaterialIndex / FirstIndex / IndexCount / WorldMatrix）
- 把命令收集阶段和命令提交阶段解耦

### `FSceneRenderer`
职责：
- 调度当前 `IRenderPath`
- 当前默认持有 `FDeferredRenderPath`
- 维护轻量级帧统计

### `FForwardRenderPath`
职责：
- 作为已落地的 Forward `IRenderPath`
- 通过 `FMeshPassProcessor(BasePass)` 收集绘制命令
- 按 PipelineKey 和 Buffer 状态排序，减少冗余绑定
- 在提交阶段通过 `FScene` 解析 `PipelineKey -> Pipeline`
- 在提交阶段通过 `FScene` 解析并绑定 BaseColor 纹理
- 在提交阶段把最多 4 个方向光和 8 个点光写入 `model.frag` 的 uniform 数组
- 提交命令到 `RHICommandBuffer`

### `FDeferredRenderPath`
职责：
- 作为当前默认 `IRenderPath`
- 管理随视口尺寸重建的 GBuffer `RHIRenderTarget`
- GBuffer Pass 使用静态网格顶点输入，输出 Albedo 与编码 WorldNormal，并写 Depth
- Lighting Pass 使用无顶点缓冲的全屏三角形，采样 GBuffer 并累加方向光与点光
- 通过 `RHIBackendTraits::bRTSampleRequiresFlipY` 显式处理 RT 采样 V 方向；当前 OpenGL 全屏三角形路径不翻转

### `FMeshPassProcessor`
职责：
- 当前只落地 BasePass
- 输入 `FScene`，输出 `FMeshDrawCommand` 列表
- 后续 GBuffer Pass 会复用同一命令构建入口

## 游戏侧到渲染侧同步

Primitive 同步过程始于 `TPrimitiveComponent`：
- 通过 `CreateSceneProxy()` 直接创建具体渲染侧代理
- `IRenderScene::AddPrimitive` 以 `FPrimitiveComponentId` 为主键注册该代理
- `FScene` 在 `AddPrimitive` 内执行资源准备并完成最终注册
- 当变换变化时标记 dirty
- `TWorld::SyncToScene()` 通过 `IRenderScene::UpdatePrimitiveTransform(id, matrix)` 同步变换

Light 同步过程始于 `LightComponent`：
- 通过 `CreateLightSceneProxy()` 创建 `FLightSceneProxy`
- `IRenderScene::AddLight` 以 `FLightComponentId` 为主键注册该代理
- 当颜色、强度、半径或变换变化时标记 dirty
- `TWorld::SyncToScene()` 通过 `IRenderScene::UpdateLight(id, proxy)` 同步光源状态

这是当前项目最关键的运行时边界之一。

## 当前还没有的内容

当前渲染器尚未实现：
- 独立渲染线程
- 运行时 Forward / Deferred 切换入口
- 可见性裁剪 Pass
- 阴影 Pass
- 完整材质系统（当前仅支持 BaseColor 纹理链路）
- 面向现代显式 API 的 Descriptor 驱动资源绑定模型
- 光源 UBO / BindGroup 批量上传（当前 Forward 多光源仍走 name-based uniform 数组）
