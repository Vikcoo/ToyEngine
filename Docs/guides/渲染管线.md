# 渲染管线

本文档描述当前代码里真实存在的渲染路径，而不是理想化设计。

## 当前渲染模型

当前渲染器是单线程、单 Pass 的简化路径，核心阶段如下：
- 游戏侧 `Primitive` 通过 `CreateSceneProxy` 直接声明具体渲染侧 `SceneProxy`
- `FScene` 维护活跃 `SceneProxy`、视图数据、以及静态网格资产级 `RenderData` 缓存
- `SceneRenderer` 从各个 Proxy 收集 `FMeshDrawCommand`
- 对 DrawCommand 做排序，减少状态切换
- 通过 `RHICommandBuffer` 提交命令
- 最终由 `RHIOpenGL` 执行实际图形 API 调用

## 关键类型

### `FScene`
职责：
- 作为 `IRenderScene` 的默认实现，提供 `Add/Update/RemovePrimitive`
- 保存当前活跃的 `FPrimitiveSceneInfo` 与 `FViewInfo`
- 维护 `TStaticMesh -> FStaticMeshRenderData` 的资产级缓存
- 维护静态网格共享 Shader/Pipeline

### `FPrimitiveSceneInfo`
职责：
- 作为 Primitive 注册到场景后的运行时节点
- 绑定 `FPrimitiveComponentId`（稳定身份）与 `FPrimitiveSceneProxy`
- 作为 `FScene` 的基础存储单元，避免场景直接持有裸 Proxy 映射

### `FStaticMeshRenderData`
职责：
- 持有资产级 GPU 数据（统一 VertexBuffer + 统一 IndexBuffer）
- 记录每个 Section 的索引范围（`FirstIndex` + `IndexCount`）
- 被多个 `FStaticMeshSceneProxy` 共享引用，避免重复上传网格数据

### `FPrimitiveSceneProxy`
职责：
- 作为游戏侧 `PrimitiveComponent` 的渲染镜像基类
- 持有实例级渲染状态（例如世界变换）
- 对外提供绘制命令生成入口

### `FStaticMeshSceneProxy`
职责：
- 只持有 `FStaticMeshRenderData` 引用与共享 Pipeline 引用
- 为各 Section 生成绘制命令
- 不直接创建或销毁 GPU 资源

### `FMeshDrawCommand`
职责：
- 打包一次绘制所需状态（Pipeline / VB / IB / FirstIndex / IndexCount / WorldMatrix）
- 把命令收集阶段和命令提交阶段解耦

### `SceneRenderer`
职责：
- 从所有活跃 Proxy 收集绘制命令
- 按 Pipeline 和 Buffer 状态排序，减少冗余绑定
- 提交命令到 `RHICommandBuffer`
- 维护轻量级帧统计

## 游戏侧到渲染侧同步

同步过程始于 `TPrimitiveComponent`：
- 通过 `CreateSceneProxy(IRenderScene&)` 直接创建具体渲染侧代理
- `IRenderScene::AddPrimitive` 以 `FPrimitiveComponentId` 为主键注册该代理
- 当变换变化时标记 dirty
- `TWorld::SyncToScene()` 通过 `IRenderScene::UpdatePrimitiveTransform(id, matrix)` 同步变换

这是当前项目最关键的运行时边界之一。

## 当前还没有的内容

当前渲染器尚未实现：
- 独立渲染线程
- 延迟渲染
- 可见性裁剪 Pass
- 阴影 Pass
- 材质图或完整材质系统
- 面向现代显式 API 的 Descriptor 驱动资源绑定模型

文档必须严格保持这一点，不能把规划内容写成现有能力。
