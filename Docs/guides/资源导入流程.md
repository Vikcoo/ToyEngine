# 资源导入流程

本文档描述当前仓库中实际存在的静态网格资源路径。

## 当前导入流程

1. `Engine::BuildScene()` 在 `Content/Models/` 中按顺序查找 `model.obj`、`model.fbx`、`model.gltf`、`model.glb`
2. `FAssetImporter::ImportStaticMesh(...)` 加载第一个成功的文件
3. 导入器把源数据转换成包含一个或多个 Section 的 `TStaticMesh`
4. `TMeshComponent` 持有该资源的 `shared_ptr`
5. `TMeshComponent::BuildRenderCreateInfo(...)` 生成渲染创建描述
6. `IRenderSceneBridge::CreatePrimitive(...)` 创建 `FStaticMeshSceneProxy`
6. Proxy 把 Section 数据上传到后端缓冲，并参与后续渲染

## 关键类型

### `TStaticMesh`
CPU 侧静态网格资源。

主要保存：
- 网格名称
- 一个或多个 `FMeshSection`

### `FAssetImporter`
引擎侧导入器入口。

职责：
- 把 Assimp 的使用范围限制在 `Asset` 模块内部
- 返回引擎自有的网格数据，而不是把 Assimp 类型暴露给其他模块

## 回退行为

如果找不到支持的外部模型文件，`Engine::CreateDefaultCubeMesh()` 会在代码中构建一个带颜色的立方体并直接使用。

这保证了即使 `Content/Models/` 中没有准备模型资源，Sandbox 仍然可以运行。

## 当前范围边界

当前资源路径刻意保持很窄：
- 以静态网格为中心
- 没有通用资源注册表
- 没有编辑器导入工作流
- 当前代码中也没有可单独说明的完整材质导入流程

文档必须只描述代码实际导入和实际使用的内容。
