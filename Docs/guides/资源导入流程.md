# 资源导入流程

本文档描述当前仓库中实际存在的静态网格资源路径。

## 当前导入流程

1. `Sandbox` 的 `SceneSetupCallback` 尝试加载 `Content/Models/orientation_cube.obj`
2. `FAssetImporter::ImportStaticMesh(...)` 加载该文件
3. 导入器把源数据转换成包含一个或多个 Section 的 `StaticMesh`
4. 导入器同步提取材质槽的最小 PBR 材质描述（BaseColor / Metallic / Roughness / Normal / AO / Emissive 的因子与贴图路径）
5. `MeshComponent` 持有该资源的 `shared_ptr`
6. `MeshComponent::CreateSceneProxy()` 仅构造带 `StaticMesh` 资产引用的 `FStaticMeshSceneProxy`
7. `IRenderScene::AddPrimitive(...)` 将该代理提交到 `FScene` 注册阶段
8. `FScene` 调用 `FRenderResourceManager` 查找/创建 `FStaticMeshRenderData`，并预热静态网格共享 Pipeline
9. `FRenderResourceManager` 为 `StaticMesh + MaterialIndex` 准备 BaseColor / Normal / Metallic / Roughness / AO / Emissive 纹理组，并按规范化路径、颜色空间和贴图语义复用 GPU 纹理
10. 资源准备完成后，`FScene` 将代理封装为 `FPrimitiveSceneInfo` 注册到场景
11. `FStaticMeshSceneProxy` 在后续渲染阶段仅引用已准备好的共享资源并生成带 `FPipelineKey` 的绘制命令
12. `SceneRenderer` 在提交阶段解析 `FPipelineKey(Pass, MaterialDomain, VertexFactory)`、绑定 Pipeline、PBR 材质纹理、MaterialBlock 与 IBL 环境资源

## 关键类型

### `FMaterial`

静态网格材质槽的最小 PBR 描述。

当前保存：
- 材质名称
- `BaseColorFactor`
- `MetallicFactor`
- `RoughnessFactor`
- `AmbientOcclusionFactor`
- `EmissiveFactor`
- `EmissiveStrength`
- BaseColor / MetallicRoughness / Metallic / Roughness / Normal / AO / Emissive 贴图槽
- 每个贴图槽的用途和颜色空间语义

当前 Renderer 已消费 BaseColor / Normal / Metallic / Roughness / AO / Emissive 贴图和材质 factor。`MetallicRoughnessTexture` 打包贴图字段已存在，但 shader 当前优先消费独立 Metallic / Roughness 贴图。

### `StaticMesh`
CPU 侧静态网格资源。

主要保存：
- 网格名称
- 一个或多个 `FMeshSection`
- 材质槽（当前为 `FMaterial` 列表）

### `FAssetImporter`
引擎侧导入器入口。

职责：
- 把 Assimp 的使用范围限制在 `Asset` 模块内部
- 返回引擎自有的网格数据，而不是把 Assimp 类型暴露给其他模块

### HDR 环境资源

当前 IBL 环境资源不是通过 `Asset` 模块导入，而是在 `Renderer` 资源准备阶段直接读取固定路径：

- `Content/Textures/HDR/citrus_orchard_road_puresky_4k.hdr`

`FRenderResourceManager` 使用 `stbi_loadf` 读取 float HDR equirectangular 图，并在运行时 CPU 生成：
- 环境 cubemap
- diffuse irradiance cubemap
- 初版 specular prefilter cubemap（当前复用环境 cubemap mip 链近似）
- BRDF LUT

这条路径是当前 IBL 的最小实现，不是完整资源系统。后续应把 HDR 环境图纳入正式资源注册表或场景级 SkyLight 配置。

## 回退行为

如果找不到支持的外部模型文件，`Sandbox` 会在代码中构建一个带颜色的默认立方体并直接使用。

这保证了即使 `Content/Models/` 中没有准备模型资源，Sandbox 仍然可以运行。

## 当前范围边界

当前资源路径刻意保持很窄：
- 以静态网格为中心
- 没有通用资源注册表
- 没有编辑器导入工作流
- 当前已有最小 PBR 材质 GPU 绑定和运行时 IBL 初版，但没有材质图、材质实例、环境资源选择、IBL 缓存落盘和编辑器资源工作流

文档必须只描述代码实际导入和实际使用的内容。

## PBR 相关规划

最小 PBR 材质描述、独立 PBR 贴图绑定和运行时 IBL 初版已经落地。后续应继续对齐 glTF 2.0 PBR 语义，把 Assimp 读取到的 BaseColor、MetallicRoughness、Normal、Occlusion、Emissive 等信息转换为引擎自己的材质描述，而不是让 Renderer 直接依赖 Assimp 材质结构。

详细前置规划见：`Docs/architecture/PBR渲染前置规划.md`。
