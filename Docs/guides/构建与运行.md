# 构建与运行

## 环境要求

- `CMake 3.18+`
- 支持项目所需标准的 C++ 编译器
- 可用的 OpenGL 运行环境
- 当前工程配置所依赖的第三方库与工具链

## 配置

常见本地配置方式：

```powershell
cmake -S . -B cmake-build-debug
```

如果你要控制当前仓库内置的编译加速选项，可以在配置时显式指定：

```powershell
cmake -S . -B cmake-build-debug `
  -DTE_ENABLE_UNITY_BUILD=ON `
  -DTE_ENABLE_PCH=ON `
  -DTE_UNITY_BUILD_BATCH_SIZE=8
```

当前新增的构建加速选项：
- `TE_ENABLE_UNITY_BUILD`：启用 Unity Build，减少编译单元数量
- `TE_ENABLE_PCH`：启用预编译头，减少重复头文件解析
- `TE_UNITY_BUILD_BATCH_SIZE`：控制 Unity Build 每批合并的源文件数量

## 构建

```powershell
cmake --build cmake-build-debug
```

## 运行

运行构建目录中生成的 Sandbox 可执行文件。实际路径取决于 CMake 生成器和构建配置。

启动时的当前行为：
- 引擎先创建窗口
- 然后尝试从 `Content/Models/` 加载模型
- 查找顺序依次为 `model.obj`、`model.fbx`、`model.gltf`、`model.glb`
- 如果都找不到，则创建一个内置立方体
- 创建飞行相机并进入主循环

## 测试

`Tests/CMakeLists.txt` 会把每个 `Tests/*.cpp` 单独编译成一个可执行文件，并且只链接 `Core` 模块。

当前含义：
- 测试较轻量，偏模块级
- 还不是完整的引擎集成测试
