# 构建与运行

## 环境要求

- `CMake 3.18+`
- 支持项目所需标准的 C++ 编译器
- 可用的 OpenGL 运行环境
- 当前工程配置所依赖的第三方库与工具链

## 配置

常见本地配置方式：

```powershell
cmake -S . -B cmake-build-release -DCMAKE_BUILD_TYPE=Release
```

当前默认约定：
- 常规开发验证、AI 构建检查、构建耗时分析默认使用 `Release`
- 默认构建目录为 `cmake-build-release`
- 只有在需要断点调试、符号排查或验证 Debug 专属问题时，才显式切换到 `cmake-build-debug`

当前仓库的构建入口仍然是标准 CMake 配置；`CMake/` 目录目前只包含平台检测、基础选项和 Shader 相关辅助脚本，还没有把 Unity Build、预编译头、编译缓存等“构建加速”能力正式接入到各个目标。

当前状态说明：
- `TE_ENABLE_UNITY_BUILD`、`TE_ENABLE_PCH`、`TE_UNITY_BUILD_BATCH_SIZE` 这些缓存项可能出现在已有构建目录中，但仓库内现有 `CMakeLists.txt` 尚未消费这些选项
- 因此是否传入这些选项，当前默认工程都不会自动为 Runtime 模块生成 Unity 源文件或预编译头
- 如果后续将这些能力正式接入，必须在同一变更中同步更新本页

## 构建

```powershell
cmake --build cmake-build-release
```

## 运行

运行构建目录中生成的 Sandbox 可执行文件。实际路径取决于 CMake 生成器和构建配置。

启动时的当前行为：
- 引擎先创建窗口
- 然后尝试从 `Content/Models/orientation_cube.obj` 加载示例模型
- 如果该文件不存在或导入失败，则创建一个内置立方体
- 创建飞行相机并进入主循环
- 调试日志每 `0.5` 秒输出平均 FPS、当前渲染相机世界坐标 `CameraWS: (X, Y, Z)` 以及 DrawCall / Pipeline / VBO / IBO 绑定统计

## 日志文件

运行 Sandbox 或调用 `TE::Log::Init()` 的测试程序后，日志会同时输出到控制台和引擎根目录下的 `Saved/Logs/ToyEngine.log`。日志目录会自动创建且不纳入 Git 管理；单个文件达到 `5 MiB` 后自动滚动，并最多保留 3 个历史文件。

程序正常退出时必须在所有日志生产线程停止、最后一条日志写入之后调用 `TE::Log::Shutdown()`，以刷新异步队列。若进程被强制终止，最近 3 秒内尚未定期刷新的普通日志仍可能丢失。

## 测试

`Tests/CMakeLists.txt` 会把每个 `Tests/*.cpp` 单独编译成一个可执行文件，并且只链接 `Core` 模块。

当前含义：
- 测试较轻量，偏模块级
- 还不是完整的引擎集成测试
