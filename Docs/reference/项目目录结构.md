# 项目目录结构

本文档给出日常开发最需要关注的仓库结构地图。

## 顶层目录

- `CMake/`：CMake 辅助模块
- `Content/`：运行时资源与 Shader 内容
- `Docs/`：项目正式文档入口
- `Scripts/`：仓库脚本
- `Source/`：引擎与 Sandbox 源码
- `Tests/`：轻量测试可执行目标
- `ThirdParty/`：内置第三方依赖
- `README.md`：仓库简要说明
- `AGENTS.md`：项目级规则与文档新鲜度约束

其中与当前构建配置直接相关的文件包括：
- `CMake/EngineOptions.cmake`：RHI 与工程级基础选项
- `CMake/PlatformDetect.cmake`：平台检测与平台宏配置
- `CMake/ShaderCompile.cmake`：Shader 相关辅助逻辑

当前仓库中尚不存在统一的 `BuildAcceleration.cmake` 或全局 `CorePCH.h`；如果未来引入统一的构建加速基础设施，需要在这里补充说明。

## Source 树

- `Source/Runtime/Core`：数学、内存、日志、通用工具
- `Source/Runtime/Platform`：窗口抽象与 GLFW 实现
- `Source/Runtime/Input`：键盘鼠标输入状态管理
- `Source/Runtime/RHI`：渲染抽象层
- `Source/Runtime/OpenGLRHI`：当前启用的后端实现
- `Source/Runtime/Asset`：静态网格存储与导入
- `Source/Runtime/Scene`：世界、Actor、组件、相机控制
- `Source/Runtime/Renderer`：渲染场景、代理对象、资产级渲染数据、渲染器
- `Source/Runtime/Engine`：引擎调度与帧循环
- `Source/Sandbox`：当前可执行入口
- `Source/Editor`：占位目录

## 文档结构

- `Docs/文档总览.md`：总入口
- `Docs/architecture/`：架构与运行流说明
- `Docs/guides/`：构建、渲染、资源等操作型说明
- `Docs/reference/`：目录结构和其他参考文档
- `Docs/API/`：未来 API 参考预留目录

## 当前文档策略

旧的平铺式专题文档已经移除，后续所有新文档都应放入上面的结构化目录中，而不是重新回到 `Docs/` 根目录堆放零散文件。
