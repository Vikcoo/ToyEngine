# 运行时模块

本文档说明当前实际参与构建的运行时模块，以及它们在项目中的职责边界。

## 当前构建顺序

`Source/Runtime/CMakeLists.txt` 当前按以下顺序加入模块：
- `Core`
- `Platform`
- `Input`
- 启用 `TE_RHI_OPENGL` 时加入 `OpenGLRHI`
- `RHI`
- `Asset`
- `Renderer`
- `Scene`
- `Engine`

这里最关键的一点是：具体后端先于抽象层 `RHI` 加入，因为 `RHI` 中的工厂代码依赖后端实现。

## 模块说明

### Core
路径：`Source/Runtime/Core`

职责：
- 数学类型与变换
- 日志系统
- 内存辅助能力
- 通用底层工具

当前接口边界：
- `Core/Public/Math` 对外暴露的是引擎自有数学类型（`Vector*`、`Matrix*`、`Quat`）
- `glm` 仅作为 `Core` 私有实现细节存在于 `Private` 中，不应出现在其他运行时模块的公开接口里

### Platform
路径：`Source/Runtime/Platform`

职责：
- 窗口抽象
- 事件轮询
- 缓冲交换与帧缓冲尺寸查询
- 输入相关回调的注册入口

当前实现：
- `Private/GLFW` 下提供基于 GLFW 的窗口实现

### Input
路径：`Source/Runtime/Input`

职责：
- 键盘状态与过渡态管理
- 鼠标按键状态管理
- 鼠标移动增量
- 滚轮增量

核心类型：
- `InputManager`

### RHI
路径：`Source/Runtime/RHI`

职责：
- 提供与图形后端无关的渲染接口
- 抽象设备、命令缓冲、缓冲区、Shader、Pipeline
- 提供公共渲染枚举与描述结构
- 提供后端工厂入口

### OpenGLRHI
路径：`Source/Runtime/OpenGLRHI`

职责：
- 为 `RHI` 接口提供 OpenGL 实现
- 作为当前实际启用的图形后端

状态：
- 当前已实现，并在开启 OpenGL 选项时参与编译

### Asset
路径：`Source/Runtime/Asset`

职责：
- 管理 CPU 侧静态网格数据
- 通过 Assimp 导入模型文件
- 组织多 Section 网格数据

核心类型：
- `TStaticMesh`
- `FAssetImporter`

### Scene
路径：`Source/Runtime/Scene`

职责：
- 管理游戏侧世界、Actor 与组件
- 维护组件变换
- 管理 Primitive 注册与脏状态
- 提供相机组件与飞行相机控制器

核心类型：
- `TWorld`
- `TActor`
- `TComponent`
- `TSceneComponent`
- `TPrimitiveComponent`
- `TMeshComponent`
- `TCameraComponent`
- `TFlyCameraController`

### Renderer
路径：`Source/Runtime/Renderer`

职责：
- 管理渲染侧场景数据
- 管理 SceneProxy 对象
- 收集并排序绘制命令
- 执行当前单 Pass 渲染提交

核心类型：
- `FScene`
- `FPrimitiveSceneProxy`
- `FStaticMeshSceneProxy`
- `FMeshDrawCommand`
- `FViewInfo`
- `SceneRenderer`

### Engine
路径：`Source/Runtime/Engine`

职责：
- 持有并初始化各子系统
- 构建 Sandbox 场景
- 驱动主循环与单帧调度
- 管理关闭顺序

核心类型：
- `Engine`

## 当前仅占位的目录

以下目录目前存在，但不能视为已实现特性：
- `Source/Runtime/VulkanRHI`
- `Source/Runtime/D3D12RHI`
- `Source/Editor`

这些目录当前更接近脚手架或保留位，而不是实际可运行模块。
