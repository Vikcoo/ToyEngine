## 这个仓库当前是什么

ToyEngine 是一个以学习引擎架构为目标的小型 C++ 项目，当前重点是通过接近 UE 风格的“游戏侧数据 / 渲染侧数据分离”来组织运行时模块。

当前实现重点：
- 使用 CMake 构建，当前可执行入口是 `Sandbox`
- CLion MinGW 下所有可执行目标默认静态链接编译器运行库，Sandbox 与测试程序无需部署 MinGW DLL 即可直接启动
- 运行时模块包括 `Core`、`Platform`、`Input`、`RHI`、`RHIOpenGL`、`Asset`、`RenderCore`、`World`、`Renderer`、`Engine`
- 当前为单线程帧循环，`Engine::Tick` 已按平台事件、输入、游戏侧更新、游戏到渲染同步、渲染阶段和帧尾收尾拆分；数据结构上已经区分游戏侧对象和渲染侧对象
- 当前启用的图形后端是 `OpenGL`，`RHIVulkan` 与 `RHID3D12` 目录仍是占位
- 通过 Assimp 导入静态模型，没有模型文件时会退化为内置立方体
- 资产侧已引入最小 `FMaterial` 材质描述，渲染侧已绑定 BaseColor / Normal / Metallic / Roughness / AO / Emissive 贴图和 `MaterialBlock`
- 当前已支持从 HDR 环境图运行时生成 IBL 初版，并在 Forward / Deferred PBR 中采样环境 cubemap、irradiance cubemap 与 BRDF LUT
- Forward / Deferred 两条路径均已支持场景方向光与点光，并支持运行时切换；当前光源批量上传已切到 `LightBlock` UBO / BindGroup
- Deferred 正式光照已使用 `Depth + inverse view-projection` 重建世界坐标；WorldPosition RT 暂时保留给 `F6/F8` 对照视图，便于观察效果并继续校准
- GPU Forward/GBuffer 已统一使用 Reversed-Z（Near=1、Far=0、清除值 0、`Greater` 比较），改善 `D32_Float` 远距离深度与位置重建精度；CPU Frustum 保持正向 RH_ZO
- 主渲染路径中的对象常量、Deferred 调试参数和主纹理采样都已迁到 UBO / BindGroup，RHI 公共命令接口已移除旧版 `SetUniform* / BindTexture2D`
- Renderer 创建 Pipeline 时只请求逻辑 Shader 名称，当前由 `RHIOpenGL` 解析到 OpenGL GLSL 资产，不再在 Renderer 中按后端选择 Shader 目录
- 当前已引入 `BindGroupLayout` / `PipelineLayout`，Renderer 区分 BindGroup group index 与 shader binding slot，为 Vulkan/D3D12 资源绑定模型预留结构基础
- Core 数学矩阵 API 已显式区分 `RH/LH` 手性与 `ZO/NO` 深度范围；当前相机默认使用 `LookAtRH()` 与 `PerspectiveRH_ZO()`
- Core 数学类型已拆分为按需包含的 `Vector.h`、`Matrix.h`、`Quat.h`；`MathTypes.h` 保持兼容聚合入口，视锥提取当前显式限定为 `RH_ZO`
- `Docs/knowledge/` 用于沉淀概念型知识，当前记录 PBR 材质参数、纹理语义与 sRGB / Linear 颜色空间基础
- 当前 `MathTest` 的旋转约定问题与 `SimpleTest` 的内存 shutdown 问题均已修复并通过 Release 验证；`MemoryShutdown` 当前采用有序停机契约，不支持与分配/释放并发调用

## 文档原则

- 文档优先描述当前已存在的代码
- 计划中的设计必须明确标识，不能伪装成既有实现
- 本文件必须长期保持为项目总索引与总目录
- 任何影响架构、行为、构建、运行、资源流的改动，都要同步更新 `Docs/`
- 实现过程中形成的技术点、优化点和工程判断也必须沉淀到 `Docs/knowledge`、`Docs/architecture` 或 `Docs/reference` 中，作为后续学习材料长期保留
